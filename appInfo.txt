▢ USER 페이지
  ◯ 라켓정보를 카드보기 형식으로 렌더링
  ◯ 사각형 카드 안에 라켓관련 다음의 정보를 제공(supabase 데이터 수신)
    - 카드보기시 가능하면 2열로 정렬하고 반응형일 것
    - 신분증 처럼 이미지와 정보 배치
    - 라켓정보 : 브랜드(brand), 이름(name), 무게(weight), 밸런스(balance), 밸런스포인트(bal_point), 탄성(flex), 스트링 최대장력(max_tension), 그립두께(grip_size), 대표색상(colors) 1~2가지
      • 영어는 무조건 대문자
    - 라켓이미지
    - 평점
      • 1인1평가
      • 평점과 참여자수만 표시 (예: ★ 4.7(26))
      • 자세히 보기: 5점/4점/3점... 별로 몇 명이 점수를 줬는지 그래프(막대기)로 표시
    - 사용자 태그
      • 라켓 별 사용자가 키언어 태그 추가 가능 (예: #초보자용 #소리좋음 #잘깨짐 #가성비)
      • 태그 등록 상위 4개 노출 및 나머지는 +5와 같이 숫자로 표시
      • 태그 하나당 최대 6자 내외로 제한
      • 자동 완성으로 태그 입력 시 기존 태그를 선택유도하여 유사 태그 분산 방지 (예: #가성비갑, #가성비최고 → #가성비)
  ◯ 라켓을 필터(브랜드, 무게, 밸런스, 탄성) 또는 태그로 검색 가능
    - 초기화면은 데이터 전송량 최소화를 위해 빈화면으로 하며 필터 선택시 렌더링

▢ 에디터 : 파이어베이스 스튜디오
▢ 프론트엔드 : vue3.js(composition API), 타입스크립트 x
▢ 백엔드 : supabase
  ◯ Auth : 구글 로그인 구현 후 다른 방식 추후 구현
  ◯ 테이블별 컬럼
    - profiles : id[uuid], nickname[text], level[text], role[text], main_racket[text], pref_weight[text], pref_balance[text], pref_position[text], gender[text]
    - rackets : id[uuid], brand[text], name[text], weight[text], balance[text], bal_point[text], flex[text], max_tension[int4], grip_size[text], colors[text[]], image_url[text]
    - tags : id[uuid], name[text]
    - racket_tag_map : racket_id[uuid], tag_id[uuid]
    - racket_reviews : id[uuid], racket_id[text], user_id[text], rating[int4]
    - ratings : id[uuid], racket_id[uuid], user_id[uuid], score[int4]
  ◯ Storage : racket_storage 폴더에 AI 라켓이미지 저장, 저장되는 파일명은 소문자+언더바

  ◯ 별점 입력
    - ratings 테이블 : id[uuid], racket_id[uuid], user_id[uuid], score[int4]
    - 수파베이스 rpc : get_racket_stats(r_id uuid)

  ◯ 라우터 사용 : admin, user, login, profile 등

▢ 스타일 : Tailwind CSS
  ◯ 빌드 후에 style이 적용이 잘 안되는 경우 tailwind 설치 관련 파일, style 관련 파일 재확인
