

▢ 배드민턴 라켓 상세정보 및 후기 제공
  ※ 프로젝트명 BDMT01
  ※ 프로젝트 구성시 확장성, 범용성 고려
  ※ 라켓 관련 모든 스펙 정보 제공
  ◯ 라켓정보를 가져와 라켓정보페이지에 여러가지 보기 형식으로 보여줌. 카드형식, 자세히보기 등 컴포넌트를 만들어 렌더링
  ◯ 사각형 카드 안에 라켓관련 다음의 정보를 제공(supabase 데이터 수신)
    - 카드보기시 2열로 정렬
    - 신분증 처럼 이미지와 정보 배치
    - 브랜드, 이름, 무게, 밸런스, 탄성, 스트링 최대장력, 그립두께, 프레임형태, 대표색상 1~2가지
    - 라켓이미지
    - 평점
      • 1인1평가
      • ★ 4.7 (26) 평점과 참여자수만 표시
      • 자세히 보기: 5점/4점/3점... 별로 몇 명이 점수를 줬는지 그래프(막대기)로 표시
    - 사용자 태그
      • 라켓 별 사용자가 키언어 태그 추가 가능 (예: #초보자용 #소리좋음 #잘깨짐 #가성비)
      • 태그 등록 상위 4개 노출 및 나머지는 +5와 같이 숫자로 표시
      • 태그 하나당 최대 6자 내외로 제한
      • 자동 완성으로 태그 입력 시 기존 태그를 선택유도하여 유사 태그 분산 방지 (예: #가성비갑, #가성비최고 → #가성비)
  ◯ 라켓을 필터(브랜드별, 무게별 등) 또는 태그로 검색 가능
    - 초기화면은 데이터 전송량 최소화를 위해 빈화면으로 하며 필터 선택시 렌더링

▢ 에디터 : 파이어베이스 스튜디오
▢ 프론트엔드 : vue3.js(composition API)
▢ 백엔드 : supabase
  ◯ 라켓별 브랜드, 길이, 무게, 밸런스, 탄성, 최대장력, 그립두께를 supabase 테이블에 저장
  ◯ supabse 라켓테이블명 : rackets, 태그테이블명 : racket_tags, 라켓-태그 관계테이블명 : rackets-tags
  ◯ 모든 장비가 공유하는 하나의 tags 테이블을 만들고, 각 장비(라켓, 신발 등)와 연결하는 매핑 테이블을 별도로 두는 방식
    - 라켓 테이블 : rackets, 공통 태그 마스터 : tags,
      라켓 전용 연결 테이블 : racket_tag_map, 셔틀콕 전용 연결 테이블 : shuttlecock_tag_map (추후)
▢ 스타일 : Tailwind CSS
  ◯ 빌드 후에 style이 적용이 잘 안되는 경우 tailwind 설치 관련 파일, style 관련 파일 재확인
