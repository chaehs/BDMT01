

▢ 배드민턴 라켓 상세정보 및 후기 제공
  ※ 모든 대화는 한국어로 할 것
  ※ 프로젝트명 BDMT01
  ※ 프로젝트 구성시 확장성, 범용성 고려
  ※ 라켓 관련 모든 스펙 정보 제공

▢ ADMIN 페이지


▢ PROFILE 페이지
  ◯ 로그인 직후 프로필 필수항목 미입력 상태인 경우 프로필 입력페이지로 이동
  ◯ 프로필 정보 : 회원등급(role), 닉네임(nickname), 성별(gender), 급수(level), 사용라켓(main_racket), 선호무게(pref_weight), 선호밸런스(pref_balance)
  ◯ 닉네임(빈칸, 나중에 DB 중복확인), 성별(M, F)
  ◯ 급수(S, AA, A, B, C, D, E, F, X)
  ◯ 사용라켓(일단 빈칸), 선호무게(3U, 4U, 5U, 6U, 7U)
  ◯ 선호밸런스(Head Heavy, Even Balance, Head Light) -> DB저장시 hh, eb, hl
  ◯ 선호포지션(Front, Back, All) -> DB저장시 front, back, all

▢ USER 페이지
  ◯ 라켓정보를 카드보기 형식으로 렌더링
  ◯ 사각형 카드 안에 라켓관련 다음의 정보를 제공(supabase 데이터 수신)
    - 카드보기시 가능하면 2열로 정렬하고 반응형일 것
    - 신분증 처럼 이미지와 정보 배치
    - 라켓카드에 스펙, 태그 모두 표기
    - 라켓 정보 : 브랜드(brand), 이름(name), 무게(weight), 밸런스(balance), 밸런스포인트(bal_point), 탄성(flex), 스트링 최대장력(max_tension), 그립두께(grip_size), 대표색상(colors) 1~2가지
      • 영어는 무조건 대문자
    - 라켓이미지
    - 평점
      • 1인1평가
      • 평점과 참여자수만 표시 (예: ★ 4.7(26))
      • 자세히 보기: 5점/4점/3점... 별로 몇 명이 점수를 줬는지 그래프(막대기)로 표시
    - 사용자 태그
      • 라켓 별 사용자가 키언어 태그 추가 가능 (예: #초보자용 #소리좋음 #잘깨짐 #가성비)
      • 태그 등록 상위 4개 노출 및 나머지는 +5와 같이 숫자로 표시
      • 태그 하나당 최대 6자 내외로 제한
      • 자동 완성으로 태그 입력 시 기존 태그를 선택유도하여 유사 태그 분산 방지 (예: #가성비갑, #가성비최고 → #가성비)
  ◯ 라켓을 필터(브랜드, 무게, 밸런스, 탄성) 또는 태그로 검색 가능
    - 초기화면은 데이터 전송량 최소화를 위해 빈화면으로 하며 필터 선택시 렌더링

▢ 에디터 : 파이어베이스 스튜디오
▢ 프론트엔드 : vue3.js(composition API), 타입스크립트 x
▢ 백엔드 : supabase
  ◯ 숫자, 영소문자, 언더바로 저장
  ◯ 모든 장비가 공유하는 하나의 tags 테이블을 만들고, 각 장비(라켓, 신발 등)와 연결하는 매핑 테이블을 별도로 두는 방식
    - 라켓 테이블 : rackets, 공통 태그 마스터 : tags, 라켓 태그 연결 테이블 : racket_tag_map, 라켓 리뷰 테이블 : racket_reviews
    - rackets 테이블 컬럼 : 브랜드(brand), 이름(name), 무게(weight), 밸런스(balance), 밸런스포인트(bal_point), 탄성(flex), 스트링 최대장력(max_tension), 그립두께(grip_size), 대표색상(colors) 1~2가지, 이미지링크(image_url)
  ◯ Auth : 구글 로그인 구현 후 다른 방식 추후 구현
    - 프로필 테이블 : profiles, 컬럼 : id[uuid], created_at[timestamptz], email[text], is_admin[bool]
  ◯ Storage : racket_storage 폴더에 AI 라켓이미지 저장, 저장되는 파일명은 소문자+언더바
  ◯ 라우터 사용 : admin, user, login

▢ 스타일 : Tailwind CSS
